# write-code skill

The **write-code** skill provides a consistent, model‑agnostic way to generate boilerplate code that follows your project's conventions. It works with OpenCode, Gemini, Kilocode, GitHub Copilot, Claude and Codex—all of which resolve skills from the `.agents/skills` directory.

## When to use
- You need to create a new component, hook, utility, or any TypeScript/JavaScript file.
- You want the generated file to respect the project's linting, formatting, and architectural patterns.
- You are working across multiple AI models/CLIs and require the same output structure regardless of the source.

## Trigger phrases
- `write code`
- `create code`
- `generate code file`

These phrases can be combined with parameters (e.g., the name of the entity) in any of the supported CLIs.

## Installation (repository‑only)
The skill lives in `.agents/skills/write-code`. No additional global installation is required; any CLI that follows the Anthropic skill‑lookup convention will automatically discover it.

## Usage examples
```bash
# OpenCode CLI
opencode use write-code --name MyComponent --type component

# Gemini CLI
gemini skill run write-code "create code MyComponent component"

# Kilocode CLI
kilocode run write-code "generate code file MyComponent.tsx"
```

### Parameters you can pass
- `--name` / entity name – the base name of the file/class/component.
- `--type` – `component`, `hook`, `utility`, etc. (determines the template used).
- Optional `--path` to override the default location inside `src/`.

## What the skill does
1. **Validates** that the target location is inside `src/`.
2. **Selects** the appropriate template based on the `type` argument.
3. **Fills** placeholders (`{{FILE_NAME}}`, `{{ENTITY_NAME}}`, `{{DESCRIPTION}}`).
4. **Adds** a JSDoc comment header describing the generated entity.
5. **Preserves** the project's import style and adds missing type imports automatically.
6. **Prompts** for clarification if an identifier is ambiguous.

## Constraints
- Files are only created within `src/`.
- Existing business‑logic files are never overwritten unless explicitly requested.
- The skill will never generate code that violates ESLint/Prettier rules.
